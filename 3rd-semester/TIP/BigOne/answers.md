Ниже — краткие ответы на каждый вопрос из списка.

1. Введение в JavaScript и базовый синтаксис  
   JavaScript — язык программирования, который широко используют для добавления интерактивности веб\-страницам (реакция на клики, анимации, динамические изменения контента).​  
   Базовый синтаксис включает литералы (числа, строки), переменные, выражения, операторы, блоки кода {}, функции, условия (if), циклы (for/while) и комментарии (//, /\* \*/).
2. Что такое JavaScript и какова его основная роль в веб\-разработке?  
   JavaScript в вебе нужен в первую очередь для интерактивности: обработка действий пользователя (клики, ввод), изменение DOM/стилей, динамическое поведение интерфейса.​  
   Он может работать и на сервере (например, через Node.js), поэтому применяется не только во фронтенде.​
3. В чем отличие JavaScript от Java?  
   JavaScript и Java — разные языки, несмотря на похожее название: у них разные цели, синтаксис, модель исполнения и экосистемы.  
   Java обычно компилируется в байткод и работает на JVM, а JavaScript чаще исполняется движком (в браузере или среде вроде Node.js) как язык сценариев.
4. Как осуществляется подключение JavaScript к HTML-документу?  
   Чаще всего JS подключают тегом \<script\>: либо пишут код прямо внутри HTML, либо подключают внешний файл через атрибут src.​  
   Дополнительно используют атрибуты defer и async, чтобы управлять моментом выполнения скрипта и не блокировать разбор HTML.
5. Какие существуют способы объявления переменных в JavaScript?  
   Основные способы: var, let, const.  
   Также технически можно создать переменную без объявления (присваиванием), но это плохая практика, потому что легко получить неявные глобальные переменные и ошибки.
6. В чем разница между var, let и const?  
   var имеет функциональную область видимости (function scope) и допускает повторное объявление в той же области.  
   let и const имеют блочную область видимости (block scope) и обычно предпочтительнее в современном коде.  
   const запрещает переназначение переменной (нельзя сделать x \= ...), но не делает автоматически неизменяемым объект внутри (содержимое объекта всё ещё можно менять).
7. Что понимается под областью видимости переменной?  
   Область видимости (scope) — это часть программы, где имя переменной доступно для чтения/изменения.  
   В JS часто выделяют глобальную область, область функции и область блока {} (для let/const).  
   JS использует лексическую (статическую) область видимости: видимость определяется тем, где код написан, а не тем, откуда он вызван.
8. Что такое выражение и чем оно отличается от оператора?  
   Выражение (expression) — фрагмент кода, который вычисляется в значение (например, 2 \+ 2, user.name, вызов функции).  
   Оператор (statement) — законченная инструкция языка, которая управляет выполнением программы (например, if (...) {...}, for (...) {...}, return ...).  
   На практике выражения часто являются частью операторов.
9. В чем отличие операторов \== и \===?  
   \== — нестрогое сравнение: перед сравнением может выполняться приведение типов (type coercion), из‑за чего возможны неожиданные результаты.  
   \=== — строгое сравнение: типы не приводятся, сравнение более предсказуемо, поэтому в большинстве случаев используют именно \===.
10. Для чего используется оператор typeof?  
    typeof возвращает строку с типом значения (например, "number", "string", "boolean", "undefined", "function", "object").  
    Его применяют для проверок типов перед операциями (валидация входных данных, защитные проверки) и для отладки.

11. **Зарезервированные слова** в JavaScript — это слова, которые имеют специальное значение в синтаксисе языка и поэтому не могут использоваться как имена переменных, функций, классов или меток (например, if, for, class, return).\[[msiter](https://msiter.ru/tutorials/javascript/js_reserved)\]​
12. Примитивные типы данных в JavaScript: string, number, bigint, boolean, undefined, symbol, null.
13. Объект в JavaScript — это ссылочный тип данных (структура), который хранит набор свойств «ключ → значение», где ключи — строки или символы, а значения могут быть любыми типами (включая функции).
14. Создать объект можно разными способами:
* Литерал: const obj \= { a: 1 }.
* Конструктор Object: const obj \= new Object().
* Object.create(proto): const obj \= Object.create(null) (или с прототипом).
* Через функцию-конструктор: function User() { ... }, затем new User().
* Через class: class User { ... }, затем new User().
15. Массив отличается от объекта тем, что массив предназначен для **упорядоченных** коллекций и обычно использует числовые индексы и свойство length, а объект — для набора именованных свойств (ключей) без акцента на порядок. На практике массив — тоже объект, но со «специализацией» под списки и с методами вроде map, filter, push.
16. undefined обычно означает «значение не задано» (например, переменная объявлена, но не инициализирована, или нет такого свойства). null — это явное присваивание «пустого/отсутствующего значения» (то есть разработчик явно указал, что значения нет).
17. Преобразование типов (type conversion / coercion) — это приведение значения к другому типу:
* Явное: Number("42"), String(10), Boolean(1).
* Неявное: при операциях и сравнениях, например "5" \* 2 (строка станет числом), или if (value) { ... } (значение приводится к boolean).
18. JSON (JavaScript Object Notation) — текстовый формат представления данных на основе объектов (пары ключ-значение) и массивов. Он используется для обмена данными (например, между клиентом и сервером в API), а также для сериализации/десериализации: JSON.stringify() и JSON.parse().
19. Оператор in проверяет, существует ли указанное свойство в объекте: 'key' in obj. Важно, что он возвращает true, даже если свойство унаследовано по прототипной цепочке (не только «собственное»).
20. Оператор delete удаляет свойство у объекта: delete obj.key. Для массивов delete arr\[1\] удаляет элемент, но обычно оставляет «дыру» (индекс становится пустым, а length не уменьшается), поэтому для удаления со сдвигом элементов чаще используют splice().

21. Symbol — это примитивный тип данных, создающий уникальные значения (даже при одинаковом описании). Символы чаще всего используют как «скрытые» ключи свойств объекта, чтобы избежать конфликтов имён и случайных перезаписей, а также для некоторых встроенных протоколов (например, итерации).
22. Условные конструкции в JavaScript включают if, else, else if и switch (часто также выделяют тернарный оператор condition ? a : b как краткую форму условия).\[[msiter](https://msiter.ru/tutorials/javascript/js_if_else_switch)\]​
23. if…else проверяет произвольные логические выражения (включая диапазоны, составные условия с &&/|| и т. п.), а switch удобен, когда нужно сравнить одно выражение с набором вариантов case и выбрать подходящую ветку. \[[msiter](https://msiter.ru/tutorials/javascript/js_if_else_switch)\]​
24. Основные виды циклов в JavaScript:
* for (классический цикл со счётчиком).
* while (проверка условия перед каждой итерацией).
* do...while (тело выполнится минимум один раз, проверка после итерации).
* for...in (перебор перечисляемых свойств объекта).
* for...of (перебор значений итерируемых коллекций: массивов, строк, Map, Set и т. д.).
25. for...in перебирает ключи (имена перечисляемых свойств) объекта и может затрагивать унаследованные перечисляемые свойства; для массивов обычно не рекомендуется. for...of перебирает именно значения и работает с итерируемыми структурами (например, массив → элементы массива), поэтому чаще подходит для коллекций.
26. Функция в JavaScript — это объект-значение, который можно вызывать, передавать как аргумент, возвращать из других функций и хранить в переменных/свойствах. Функции используются для переиспользования кода, инкапсуляции логики и построения абстракций.
27. Декларация функции (Function Declaration) создаёт именованную функцию как отдельную конструкцию: function foo() {}; такие функции обычно доступны до места объявления (поднимаются). Функциональное выражение (Function Expression) — это когда функция создаётся как значение выражения: const foo \= function() {}; оно становится доступным только после выполнения строки с присваиванием.
28. Стрелочные функции — сокращённый синтаксис функций: const f \= (a) \=\> a \* 2. Ключевые отличия: у них нет собственного this (берут this из внешней области видимости) и их нельзя использовать как конструктор с new.
29. Параметры по умолчанию — это значения, которые подставляются, если аргумент не передали или передали undefined:  
    function greet(name \= "Гость") { ... }.
30. Рекурсия — это приём в программировании, когда функция вызывает саму себя для решения задачи через более простые подзадачи. Обычно нужна «база рекурсии» (условие остановки), иначе вызовы будут продолжаться бесконечно.

31. **Функция — это объект**, потому что в JavaScript функции являются специальным типом объектов (наследуют от `Function.prototype`), могут иметь свойства (например, `name`, `length`), методы (такие как `call`, `bind`) и передаваться как значения в переменные или аргументы.\[[developer.mozilla](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Working_with_objects)\]​
32. При арифметических операциях (кроме `+`), если операнды не являются числами, интерпретатор пытается привести их к числу (например, `"5" * 2` → `10`). Оператор `+` работает иначе: если хотя бы один операнд — строка, то происходит конкатенация (склеивание) строк, а не сложение (например, `5 + "5"` → `"55"`).\[[developer.mozilla](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Data_structures)\]​
33. **Динамическая типизация** означает, что типы данных привязаны к значениям, а не к переменным. Одна и та же переменная может в разное время хранить значения разных типов (сначала число, затем строку и т.д.), и проверка типов происходит во время выполнения программы.\[[arxiv](http://arxiv.org/pdf/2105.13699.pdf)\]​
34. **Массивы** в JavaScript — это упорядоченные коллекции, которые имеют свойство `length` (автоматически обновляемое при изменении индексов) и специальные методы для работы с данными (`push`, `pop`, `map`). Они могут содержать элементы разных типов и технически являются объектами с числовыми ключами.\[[shapkarin](https://shapkarin.me/articles/curiosity-js/)\]​
35. **Неявное приведение типов** (coercion) — это автоматическое преобразование значения из одного типа в другой, которое выполняет интерпретатор при операциях сравнения (`==`), арифметических действиях или логических проверках (например, `if ("")` воспринимается как `false`).\[[uzma.hashnode](https://uzma.hashnode.dev/coercion-from-basics)\]​
36. Массивы считаются **частным случаем объектов**, так как тип массива (`typeof []`) — это `'object'`, они передаются по ссылке и используют внутреннюю структуру хеш-таблицы (ключ-значение), где ключами выступают индексы (строки "0", "1" и т.д.), но имеют специальный прототип `Array.prototype`.\[[exploringjs](https://exploringjs.com/deep-js/ch_type-coercion.html)\]​
37. **DOM (Document Object Model)** — это программный интерфейс для HTML и XML документов, представляющий структуру страницы в виде дерева объектов. Его роль заключается в том, чтобы дать скриптам (например, JavaScript) возможность менять содержимое, стили и структуру веб\-страницы динамически.\[[developer.chrome](https://developer.chrome.com/blog/devtools-tips-28)\]​
38. **Отличие DOM от HTML-документа**: HTML — это исходный текстовый код (разметка), который приходит с сервера, а DOM — это объектное представление этой разметки в памяти браузера, которое может отличаться от исходного HTML (например, браузер может автоматически исправить ошибки разметки или скрипты могут изменить DOM после загрузки).\[[guvi](https://www.guvi.in/blog/guide-for-html-dom/)\]
39. Способы **поиска элементов в DOM**:
* `document.getElementById('id')` — поиск по ID.
* `document.querySelector('css-selector')` — поиск первого элемента по CSS-селектору.
* `document.querySelectorAll('css-selector')` — поиск всех элементов по CSS-селектору.
* `document.getElementsByClassName('class')` — поиск по классу (возвращает живую коллекцию).
* `document.getElementsByTagName('tag')` — поиск по тегу.
40. **CSS-селекторы** — это шаблоны для выбора элементов (например, `.class`, `#id`, `div > p`). В DOM-методах (таких как `querySelector`) они применяются для того, чтобы точно и гибко находить элементы, используя тот же синтаксис, что и в CSS-файлах для стилизации.\[[developer.chrome](https://developer.chrome.com/blog/devtools-tips-28)\]​

41. Отличие `querySelector` от `getElementById`:
* **`getElementById`** ищет элемент только по его атрибуту `id` (например, `'#myId'`), он немного быстрее, но работает только с ID и вызывается только на самом `document`.
* **`querySelector`** более универсален: он принимает любой валидный CSS-селектор (ID, класс, атрибуты, сложные вложенности: `div > .item[name="x"]`). Он возвращает *первый* найденный элемент и может вызываться на любом элементе DOM, а не только на `document`.\[[ru.stackoverflow](https://ru.stackoverflow.com/questions/995930/%D0%92-%D1%87%D1%91%D0%BC-%D1%80%D0%B0%D0%B7%D0%BD%D0%B8%D1%86%D0%B0-%D0%BC%D0%B5%D0%B6%D0%B4%D1%83-queryselector-%D0%B8-getelementbyid)\]
42. Свойство **`innerHTML`** позволяет получить или изменить HTML-содержимое элемента в виде строки. При записи в него браузер парсит строку и пересоздаёт соответствующие DOM-узлы внутри элемента.
43. Разница между `innerHTML` и `textContent`:
* **`innerHTML`** возвращает содержимое вместе с тегами (HTML-разметкой) и при записи интерпретирует теги.
* **`textContent`** работает только с «чистым» текстом. При чтении он возвращает текст всех потомков (без тегов), а при записи вставляет строку как безопасный текст (теги экранируются и выводятся как символы, а не превращаются в элементы), что безопаснее для защиты от XSS-атак.
44. JavaScript может изменять **структуру DOM** (Document Object Model) через методы создания, добавления и удаления узлов. Скрипты могут создавать новые элементы (`document.createElement`), вставлять их в нужное место (`append`, `prepend`, `before`, `after`) и менять иерархию существующих элементов.
45. Основные методы **удаления элементов**:
* `element.remove()` — современный метод, удаляет сам элемент, на котором вызван.
* `parent.removeChild(child)` — старый метод, удаляет дочерний узел `child` из родителя `parent`.
46. Работа с **атрибутами** осуществляется через методы:
* `getAttribute('name')` — получить значение.
* `setAttribute('name', 'value')` — установить значение.
* `removeAttribute('name')` — удалить атрибут.
* `hasAttribute('name')` — проверить наличие.  
  Также многие стандартные атрибуты доступны как прямые свойства объекта DOM (`input.value`, `img.src`, `a.href`), но их поведение может немного отличаться от атрибутов в HTML.
47. **Событие** (event) — это сигнал от браузера о том, что что-то произошло: пользователь кликнул мышью, нажал клавишу, страница загрузилась, изменился размер окна и т.д. JavaScript позволяет «слушать» эти сигналы и реагировать на них.
48. Способы **привязки обработчиков**:
* Через HTML-атрибут (не рекомендуется): `<button onclick="func()">`.
* Через свойство DOM-объекта: `elem.onclick = function() { ... }`.
* Через метод `addEventListener`: `elem.addEventListener('click', handler)`.
49. Отличие **`onclick`** от **`addEventListener`**:
* Свойство `onclick` позволяет назначить *только один* обработчик на событие (новое присваивание перезапишет старое).
* `addEventListener` позволяет вешать *несколько* обработчиков на одно и то же событие (они сработают по очереди), а также дает возможность управлять фазами (всплытие/перехват) и удалять обработчики через `removeEventListener`.
50. **Объект события (event)** — это объект, который автоматически передаётся первым аргументом в функцию-обработчик. Он содержит детали произошедшего события: тип события (`type`), цель (`target` — элемент, где событие возникло), координаты курсора, нажатые клавиши и методы управления событием (например, `preventDefault()` для отмены стандартного действия браузера).

**51\. Предотвращение стандартного действия события**  
Предотвращение стандартного действия события означает отмену поведения браузера, которое обычно происходит при возникновении события. Это реализуется с помощью метода preventDefault(), вызываемого на объекте события. Например, при клике на ссылку браузер не будет переходить по URL, а при отправке формы страница не перезагрузится.​

## **52\. Взаимодействие JavaScript с элементами форм**

JavaScript взаимодействует с элементами форм через DOM API, получая доступ к полям ввода, кнопкам и другим элементам с помощью методов типа querySelector() или getElementById(). После получения доступа к элементу можно читать и изменять его значения через свойства (например, value для текстовых полей), добавлять обработчики событий для отслеживания действий пользователя. Также можно программно отправлять формы, валидировать данные и предотвращать отправку через preventDefault().​

## **53\. Всплытие событий**

Всплытие событий (event bubbling) — это механизм распространения событий в DOM от целевого элемента вверх по дереву элементов к его родителям. Когда происходит событие на элементе, оно сначала обрабатывается на самом элементе, затем на его родителе, затем выше и так далее до корня документа. Этот механизм позволяет перехватывать события на родительских элементах, что используется в делегировании событий.​

## **54\. Остановка всплытия события**

Остановка всплытия события означает прекращение дальнейшего распространения события вверх по дереву DOM. Это достигается вызовом метода stopPropagation() на объекте события, который "протыкает пузырь" и не позволяет событию достичь родительских элементов. В отличие от preventDefault(), который отменяет стандартное действие браузера, stopPropagation() влияет только на распространение события между элементами.​

## **55\. Делегирование событий**

Делегирование событий — это техника, при которой обработчик события назначается не на каждый отдельный элемент, а на их общего родителя. Когда событие всплывает от дочернего элемента к родителю, обработчик проверяет источник события и выполняет нужное действие. Это позволяет эффективно обрабатывать события для множества элементов с помощью одного обработчика, особенно полезно для динамически добавляемых элементов.​

## **56\. Различие событий input и change**

Событие input срабатывает немедленно при каждом изменении значения элемента формы, то есть при каждом нажатии клавиши в текстовом поле. Событие change срабатывает только после того, как элемент теряет фокус и его значение было изменено с момента получения фокуса. Для текстовых полей change активируется после завершения редактирования, а для чекбоксов и радиокнопок — сразу при изменении состояния.​

## **57\. Различие атрибутов defer и async**

Атрибут defer указывает браузеру загружать скрипт параллельно с HTML, но выполнять его только после полного построения DOM, при этом скрипты с defer выполняются в порядке их объявления. Атрибут async также загружает скрипт асинхронно, но выполняет его сразу после загрузки, независимо от состояния DOM и без гарантии порядка выполнения относительно других скриптов. Оба атрибута позволяют избежать блокировки парсинга HTML во время загрузки скрипта.​

## **58\. Событие DOMContentLoaded**

Событие DOMContentLoaded возникает, когда браузер полностью загрузил и построил HTML-документ, и дерево DOM готово к использованию. Это событие срабатывает до загрузки внешних ресурсов, таких как изображения, стили и фреймы. Обработчики этого события часто используются для инициализации скриптов, которым нужен доступ к элементам DOM, но не требуется ждать загрузки всех ресурсов страницы.​

## **59\. Отличие DOMContentLoaded от события load**

Событие DOMContentLoaded срабатывает, когда построено только дерево DOM, но внешние ресурсы (изображения, стили, фреймы) еще могут загружаться. Событие load на объекте window срабатывает позже — когда полностью загружены все ресурсы страницы, включая изображения и стили. DOMContentLoaded позволяет начать работу со страницей раньше, не дожидаясь загрузки тяжелых ресурсов.​

## **60\. Блокировка рендеринга при загрузке скриптов**

Блокировка рендеринга означает, что браузер приостанавливает построение DOM и отображение страницы, пока загружается и выполняется синхронный скрипт без атрибутов defer или async. Это происходит потому, что скрипт может изменить структуру документа через document.write() или другие методы. Для предотвращения блокировки рекомендуется размещать скрипты в конце \<body\> или использовать атрибуты defer/async.​

## **61\. Порядок выполнения скриптов с атрибутом async**

Скрипты с атрибутом async загружаются параллельно и выполняются в непредсказуемом порядке — браузер запускает их сразу после загрузки, независимо от порядка объявления в HTML. Не существует предсказуемого порядка между несколькими асинхронными скриптами, так как браузер свободен выполнять их в любой последовательности. Если требуется определённый порядок выполнения, необходимо использовать обработчики событий загрузки и вручную управлять последовательностью вызовов.​

## **62\. Дерево элементов документа и его формирование**

Дерево элементов документа (DOM tree) — это иерархическая структура, представляющая HTML-документ в виде узлов, где каждый элемент, атрибут и текст являются отдельными узлами. Формирование происходит в процессе парсинга HTML: браузер читает разметку сверху вниз, создавая узлы и устанавливая связи родитель-потомок согласно вложенности элементов. После построения дерева DOM становится доступным для манипуляций через JavaScript API.​

## **63\. Отличие изменения структуры DOM от изменения содержимого**

Изменение структуры DOM включает добавление, удаление или перемещение элементов в дереве документа с помощью методов типа appendChild(), removeChild(), insertBefore(). Изменение содержимого означает модификацию текста или HTML внутри существующих элементов через свойства textContent, innerHTML или innerText без изменения самой структуры узлов. Структурные изменения влияют на количество и расположение элементов, тогда как изменение содержимого затрагивает только данные внутри неизменных узлов.​

## **64\. Способы управления стилями элементов через JavaScript**

JavaScript позволяет управлять стилями через свойство style элемента, которое даёт доступ к inline-стилям (например, element.style.color \= 'red'). Второй способ — манипуляция CSS-классами через свойства className или методы classList.add(), classList.remove(), classList.toggle(). Также можно динамически создавать или модифицировать элементы \<style\> и таблицы стилей через CSSOM API.​

## **65\. Управление видимостью элементов в DOM**

Видимостью элементов можно управлять через CSS-свойство display, устанавливая значения none (скрывает элемент полностью) или block/inline (показывает элемент). Альтернативный способ — использование свойства visibility со значениями hidden (скрывает элемент, но сохраняет его место) или visible. Также применяется свойство opacity для управления прозрачностью от 0 (невидимый) до 1 (полностью видимый).​

## **66\. Методы получения размеров окна браузера и элементов**

Размеры окна браузера получают через свойства window.innerWidth и window.innerHeight (включая полосы прокрутки) или document.documentElement.clientWidth и clientHeight (без полос прокрутки). Для элементов используются свойства offsetWidth/offsetHeight (полные размеры с границами), clientWidth/clientHeight (размеры содержимого с padding, но без границ и прокрутки). Метод getBoundingClientRect() возвращает объект с координатами и размерами элемента относительно viewport.​

## **67\. Замыкание в JavaScript**

Замыкание (closure) — это функция, которая имеет доступ к переменным из своей внешней (родительской) области видимости, даже после того, как внешняя функция завершила выполнение. Замыкания создаются автоматически каждый раз, когда функция создаётся внутри другой функции, сохраняя ссылку на лексическое окружение. Это позволяет внутренней функции «запомнить» окружение, в котором она была создана, и использовать эти переменные позже.​

## **68\. Отличие именованных и анонимных функций**

Именованные функции имеют идентификатор (имя) после ключевого слова function, что позволяет вызывать их по имени и видеть это имя в стеке вызовов при отладке. Анонимные функции не имеют имени и обычно используются как функциональные выражения, присваиваемые переменным, или как callback-функции. Именованные функции можно вызывать рекурсивно по имени внутри себя, тогда как анонимные функции требуют для этого ссылку на переменную.​

## **69\. Роль ключевого слова this в JavaScript**

Ключевое слово this в JavaScript ссылается на контекст выполнения функции и определяет, к какому объекту принадлежит текущий код. Значение this зависит от способа вызова функции: в методах объекта это сам объект, в обычных функциях — глобальный объект (или undefined в строгом режиме), в конструкторах — новый создаваемый объект. С помощью методов call(), apply() и bind() можно явно устанавливать значение this.​

## **70\. Разница между методами call, apply и bind**

Метод call() немедленно вызывает функцию с указанным контекстом this, принимая аргументы по отдельности после контекста. Метод apply() также немедленно вызывает функцию с заданным this, но принимает аргументы в виде массива или массивоподобного объекта. Метод bind() не выполняет функцию сразу, а создаёт и возвращает новую функцию с фиксированным значением this, которую можно вызвать позже.​

## **71\. Геттеры и сеттеры в объектах**

Геттеры и сеттеры — это специальные методы объекта, которые позволяют контролировать доступ к свойствам. Геттер (метод get) срабатывает при чтении свойства и позволяет вычислить возвращаемое значение динамически, а сеттер (метод set) вызывается при попытке записи значения и позволяет выполнить валидацию или преобразование данных перед их сохранением. Внешне работа с такими свойствами выглядит как обращение к обычной переменной, но внутри выполняется код функций.​

## **72\. Дескриптор свойства объекта**

Дескриптор — это объект, который описывает поведение свойства и содержит его настройки (флаги). Он может быть двух типов: дескриптор данных (содержит value и флаг writable) или дескриптор доступа (содержит методы get и set). Основные флаги дескриптора включают enumerable (видимость в циклах), configurable (возможность удаления или изменения дескриптора) и writable (возможность изменения значения).​

## **73\. Функции высшего порядка**

Функции высшего порядка — это функции, которые принимают другие функции в качестве аргументов или возвращают функции как результат своего выполнения. Примерами встроенных функций высшего порядка в JavaScript являются методы массивов map, filter, reduce, а также функция addEventListener, принимающая callback-функцию. Этот механизм лежит в основе функционального программирования и позволяет создавать абстракции и компонуемые блоки кода.​

## **74\. Каррирование функций**

Каррирование (currying) — это трансформация функции с множеством аргументов в последовательность вложенных функций, каждая из которых принимает только один аргумент. Вместо вызова func(a, b, c) каррированная функция вызывается как func(a)(b)(c). Это позволяет создавать частично примененные функции, фиксируя некоторые аргументы заранее и переиспользуя полученную функцию с оставшимися параметрами.​

## **75\. Конструкция IIFE (Immediately Invoked Function Expression)**

IIFE — это выражение функции, которое определяется и немедленно выполняется сразу после создания. Синтаксически это выглядит как функция, обернутая в круглые скобки, за которыми следуют вызывающие скобки: (function() { ... })();. Основное назначение IIFE — создание изолированной области видимости для переменных, чтобы они не загрязняли глобальное пространство имен, что было особенно актуально до появления блочной области видимости let и const.​

## **76\. Роль метода Object.freeze**

Метод Object.freeze() "замораживает" объект, делая его полностью неизменяемым. После применения этого метода к объекту нельзя добавлять новые свойства, удалять существующие, изменять их значения или настройки дескрипторов (флаги writable, configurable становятся false). Попытка изменить замороженный объект в строгом режиме (use strict) приведет к ошибке, а в обычном режиме просто не сработает.​

## **77\. Объект Promise**

Promise (промис) — это объект, представляющий собой результат асинхронной операции, который может быть получен в будущем. Он служит контейнером для значения, которое пока неизвестно, но будет известно позже (успешный результат или ошибка). Промис позволяет писать асинхронный код более линейно и чисто, избегая глубокой вложенности колбэков ("callback hell").​

## **78\. Разница между состояниями Promise**

У промиса есть три возможных состояния: pending (ожидание) — исходное состояние, когда операция еще не завершена; fulfilled (выполнено) — операция успешно завершилась и вернула результат; rejected (отклонено) — произошла ошибка. Промис может перейти из pending только в одно из завершающих состояний (fulfilled или rejected) и только один раз — после этого состояние становится неизменным (settled).​

## **79\. Цепочка промисов**

Цепочка промисов (promise chaining) — это последовательное выполнение асинхронных операций, где результат одной операции передается следующей через методы .then(). Каждый вызов .then() возвращает новый промис, что позволяет выстраивать длинные последовательности действий. Если в цепочке возникает ошибка, управление передается ближайшему обработчику .catch(), пропуская остальные .then().​

## **80\. Назначение метода Promise.all**

Метод Promise.all() принимает массив (или итерируемый объект) промисов и возвращает новый промис, который выполняется только тогда, когда успешно завершаются *все* переданные промисы. Результатом становится массив значений всех выполненных промисов в том же порядке. Если хотя бы один из переданных промисов завершится с ошибкой (rejected), то весь Promise.all() немедленно завершится с этой же ошибкой, игнорируя результаты остальных.

## **81\. Преимущества использования async/await**

Использование async/await позволяет писать асинхронный код, который выглядит и ведет себя как синхронный, что значительно упрощает чтение и отладку. Основные преимущества включают избавление от "адского колбэка" (callback hell) и цепочек .then(), что делает код более линейным и понятным. Также async/await упрощает обработку ошибок с помощью стандартной конструкции try/catch и позволяет использовать обычные циклы и условные операторы для управления потоком асинхронных операций.​

## **82\. Обработка ошибок при использовании async/await**

В функциях с async/await ошибки обрабатываются с помощью синхронной конструкции try...catch. Весь код, который может вызвать ошибку (например, await promise), оборачивается в блок try, а обработка исключений происходит в блоке catch(error). Если промис будет отклонен (rejected), выполнение в блоке try прервется, и управление перейдет в catch, где можно логировать ошибку или показать сообщение пользователю.​

## **83\. Различие функций setTimeout и setInterval**

Функция setTimeout позволяет выполнить код один раз через заданный промежуток времени. Функция setInterval, напротив, запускает код многократно через регулярные интервалы времени, пока не будет остановлена с помощью clearInterval. В контексте Event Loop оба этих метода создают макрозадачи.​

## **84\. Цикл событий (Event Loop)**

Цикл событий (Event Loop) — это механизм, который координирует выполнение кода, событий и подзадач в JavaScript. Он следит за стеком вызовов (Call Stack) и очередями задач: когда стек пуст, Event Loop берет задачи из очередей (сначала микрозадачи, затем одну макрозадачу) и помещает их в стек для выполнения. Это позволяет JavaScript быть однопоточным, но при этом выполнять асинхронные операции без блокировки интерфейса.​

## **85\. Микрозадачи и макрозадачи в Event Loop**

Микрозадачи (microtasks) имеют более высокий приоритет и включают промисы (Promise.then, catch, finally) и MutationObserver. Макрозадачи (macrotasks) включают таймеры (setTimeout, setInterval), события I/O и UI-рендеринг. В каждом цикле Event Loop сначала выполняются *все* накопившиеся микрозадачи до тех пор, пока их очередь не станет пустой, и только после этого выполняется *одна* макрозадача из очереди.​

## **86\. Назначение метода fetch**

Метод fetch используется для выполнения сетевых запросов (HTTP-запросов) к серверам для получения или отправки данных. Он возвращает промис, который разрешается объектом Response, предоставляющим методы для чтения ответа (например, .json(), .text()). fetch является современной заменой устаревшему XMLHttpRequest и предоставляет более мощный и гибкий API для работы с сетью.​

## **87\. Библиотека React и её использование**

React — это JavaScript-библиотека для создания пользовательских интерфейсов, разработанная Facebook. Она используется для построения SPA (Single Page Applications) и интерактивных веб\-приложений, где интерфейс разбивается на независимые переиспользуемые части — компоненты. React использует виртуальный DOM (Virtual DOM) для эффективного обновления только тех частей страницы, которые изменились, что повышает производительность.​

## **88\. Компонент в React**

Компонент в React — это изолированный, независимый блок кода, который отвечает за отображение части интерфейса (UI). Компоненты могут принимать входные данные (props) и возвращать React-элементы, описывающие, что должно появиться на экране. Они позволяют разбивать сложный UI на простые, переиспользуемые части, которые можно разрабатывать и тестировать отдельно.​

## **89\. Отличие функциональных и классовых компонентов**

Функциональные компоненты — это обычные JavaScript-функции, которые проще в написании и используют хуки (hooks) для работы с состоянием и эффектами. Классовые компоненты — это классы ES6, которые наследуются от React.Component и используют метод render(), а также имеют методы жизненного цикла (например, componentDidMount) и объект this.state для управления состоянием. В современной разработке (с 2019 года) функциональные компоненты с хуками являются стандартом.​

## **90\. Состояние (state) в React**

Состояние (state) — это объект, содержащий данные, которые могут изменяться в течение жизни компонента и влиять на его отображение. При изменении состояния React автоматически перерисовывает компонент, чтобы интерфейс соответствовал новым данным. В функциональных компонентах состояние управляется хуком useState, а в классовых — через this.state и this.setState().​

## **91\. Разница между props и state**

В React props (свойства) — это данные, которые передаются от родительского компонента к дочернему и доступны только для чтения (неизменяемы). state (состояние) — это локальные данные компонента, которые управляются внутри него, могут изменяться со временем (мутабельны) и при их изменении вызывают перерисовку (ре-рендер) компонента. props используются для настройки компонента извне, а state — для управления его внутренним поведением и реакцией на действия пользователя.​

## **92\. Обновление состояния в React**

В функциональных компонентах состояние обновляется с помощью функции, возвращаемой хуком useState (например, const \[count, setCount\] \= useState(0)), где вызов setCount(newValue) обновляет значение и вызывает рендер. В классовых компонентах используется метод this.setState(), который принимает объект с обновляемыми полями и асинхронно вносит изменения в this.state, после чего также происходит ре-рендер.​

## **93\. Подъем состояния (state lifting)**

Подъем состояния (lifting state up) — это паттерн, при котором состояние перемещается из дочерних компонентов в их общего родителя (предка). Это необходимо, когда нескольким компонентам (обычно "братьям") нужно использовать одни и те же данные или синхронизироваться между собой. Родитель управляет состоянием и передает его значения и функции для их изменения своим потомкам через props.​

## **94\. Рендеринг списков в React**

Рендеринг списков в React обычно осуществляется с помощью метода массива .map(), который преобразует массив данных в массив JSX-элементов. Внутри функции .map() для каждого элемента данных возвращается компонент или HTML-тег, который React затем отображает по порядку.​

## **95\. Использование атрибута key при рендеринге списков**

Атрибут key — это специальная строковая пропса, которая помогает React идентифицировать, какие элементы списка были изменены, добавлены или удалены. Использование уникальных и стабильных ключей (обычно ID из базы данных) критически важно для производительности и корректного обновления интерфейса при изменении порядка элементов, так как это позволяет React переиспользовать существующие DOM-узлы вместо их полной перерисовки.​

## **96\. Работа с формами в React**

В React работа с формами чаще всего реализуется через "управляемые компоненты" (controlled components), где значением поля ввода управляет состояние React через проп value, а любые изменения отслеживаются через обработчик onChange. При каждом вводе символа вызывается функция обновления состояния, и новое значение передается обратно в input, обеспечивая полный контроль над данными формы и их валидацию в реальном времени.​

## **97\. Node.js и его особенности**

Node.js — это кроссплатформенная среда выполнения JavaScript, построенная на движке V8 (от Google Chrome), позволяющая запускать JS-код вне браузера, обычно на сервере. Его главные особенности: событийно-ориентированная архитектура (event-driven), использование неблокирующего ввода-вывода (non-blocking I/O) и однопоточный цикл событий (Event Loop), что делает его эффективным для создания масштабируемых сетевых приложений с высокой нагрузкой.​

## **98\. Отличие Node.js от JavaScript в браузере**

В Node.js отсутствуют объекты браузера, такие как window и document (DOM API), так как нет графического интерфейса. Вместо этого Node.js предоставляет модули для работы с файловой системой (fs), операционной системой и сетью, а глобальным объектом является global. Также в Node.js используется модульная система CommonJS (require/module.exports) по умолчанию, хотя поддержка ES-модулей также доступна.​

## **99\. Модули Node.js для работы с файлами**

Для работы с файлами в Node.js используется встроенный модуль fs (File System). Он предоставляет методы для чтения (fs.readFile), записи (fs.writeFile), добавления данных (fs.appendFile), удаления файлов (fs.unlink) и работы с директориями, причем большинство методов имеют как асинхронные, так и синхронные (с суффиксом Sync) версии.​

## **100\. Принцип работы HTTP-сервера в Node.js**

HTTP-сервер в Node.js создается с помощью модуля http и метода createServer(), который принимает функцию-обработчик (callback). Этот обработчик вызывается каждый раз при поступлении запроса и получает два объекта: request (информация о запросе) и response (объект для отправки ответа). Сервер "слушает" определенный порт методом .listen(), ожидая входящие соединения и запуская обработчик для каждого из них асинхронно.​

## **101\. Express.js и решаемые им задачи**

Express.js — это минималистичный и гибкий веб\-фреймворк для Node.js, который упрощает создание веб\-приложений и API. Он решает задачи маршрутизации (routing) запросов по URL и методам HTTP, предоставляет удобный механизм промежуточного ПО (middleware) для обработки запросов (например, логирование, аутентификация), упрощает работу с запросами/ответами и интеграцию с шаблонизаторами, избавляя от написания большого количества шаблонного кода на чистом Node.js.​
